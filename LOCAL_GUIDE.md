# دليل التطوير المحلي والرفع لـ Railway

هذا الدليل يشرح كيف يمكنك تعديل البرنامج على جهازك وكيف نقوم برفع التعديلات معاً.

## 1. كيف نتعامل معاً محلياً؟
بما أنني Antigravity وموجود داخل محررك، فنحن **نتعامل محلياً بالفعل**.
- أي ملف تطلبه مني، أنا أراه وأعدله مباشرة في المجلد الموجود على جهازك: `c:\Users\dell\Desktop\gold`.
- يمكنك إجراء أي تعديل بيدك، وسأراه أنا في الخطوة التالية.

## 2. خطوات العمل (المسار الصحيح)

### الخطوة أ: التعديل والبرمجة
نحن نقوم بهذه الخطوة الآن. نعدل ملفات الـ `templates` أو الـ `views` حتى نصل للنتيجة المطلوبة.

### الخطوة ب: المراجعة
بعد التعديل، يمكنك مراجعة الملفات في VS Code.

### الخطوة ج: الرفع (Deployment)
عندما نتفق على أن العمل انتهى، نستخدم الأوامر التالية لنقل التعديلات من "جهازك" إلى "الإنترنت":

```powershell
# 1. إضافة كل الملفات المعدلة
git add .

# 2. تسجيل التعديلات بوصف مختصر
git commit -m "وصف لما قمنا بفعله"

# 3. الرفع لـ GitHub (سيقوم Railway بالتحديث تلقائياً)
git push origin main
```

## 3. نصيحة هامة
دائماً اطلب مني `git status` قبل الرفع لنتأكد من الملفات التي سيتم رفعها.
